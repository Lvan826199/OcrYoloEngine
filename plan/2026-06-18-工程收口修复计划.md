# 2026-06-18 工程收口修复计划

## 背景

本批次针对 v0.2.1 后审阅发现的工程边界问题做集中修复。目标不是新增大功能，而是让缓存、错误契约、Docker 开箱体验、配置校验、上传接口与 Windows 路径白名单更贴近已有文档承诺。

## 问题清单

| 优先级 | 问题 | 位置 | 修法 | 验证 |
|---|---|---|---|---|
| P0 | 结果缓存键对 `methods` 排序，`merge=priority` 的顺序语义可能串缓存 | `src/ocr_yolo_engine/cache.py` | 缓存键保留 `methods` 与 `templates` 原始顺序 | 单测断言不同顺序生成不同 key |
| P0 | Docker 示例模板路径指向 `tests/fixtures/golden_patch.png`，镜像内未复制该资产 | `docker/Dockerfile.*`、`.dockerignore` | 根目录新增 `.dockerignore` 并仅放行 demo fixture；Dockerfile 显式复制小体积 demo 图 | 单测检查 Dockerfile 与根 `.dockerignore` 闭环 |
| P1 | 鉴权失败与请求校验失败未走统一错误契约 | `errors.py`、`service/app.py`、`service/auth.py` | 新增 `UNAUTHORIZED`、`VALIDATION_ERROR`，统一返回 `ErrorResponse` 结构 | 契约测试断言 401/422 均含 `error_code` 与真实 `request_id` |
| P1 | 运行配置缺少边界约束，错误环境变量可能延迟到运行期炸 | `settings.py` | 用 Pydantic `Field` 约束阈值、缓存、并发、超时、输入上限 | 单测断言非法配置在加载期拒绝 |
| P1 | 文件上传接口与 `/v1/recognize` 字段不齐 | `service/routes.py` | multipart 补 `roi`、`debug`、`cache`、`merge` 字段，`roi` 用 JSON 字符串 | 真实 multipart 契约测试覆盖 ROI、debug、merge、cache |
| P2 | Windows 下白名单路径大小写差异可能误拒合法路径 | `image/loader.py` | 路径比较前使用 `normcase(realpath)` | Windows 条件单测覆盖大小写不同的白名单 |

## 批次步骤

1. 先补上述单测 / 契约测试，确认当前实现暴露问题。
2. 小步修改实现与 Docker 构建文件。
3. 同步 `docs/bug修复日志.md`、`docs/开发说明.md`、`docs/使用文档.md`、`docs/接口集成指南.md`、`docs/部署文档.md`、`CHANGELOG.md`。
4. 跑 `ruff`、`mypy`、默认测试与覆盖率门禁，全绿后提交推送。
