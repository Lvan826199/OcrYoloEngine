## 1. 一句话定位

`OcrYoloEngine` 是面向**自动化测试**的视觉识别 HTTP 服务:其他自动化脚本截图发来,本服务用 OCR / YOLO / 模板匹配识别,返回坐标、文字、置信度,**只识别、不执行动作**。

- **debug 标注图**:`debug=true` 返回画框 PNG(base64)。

- **golden 回归**:`tests/fixtures/` 提交确定性样例图 + 期望结果。
- **持续集成**:`scripts/check.sh`、`Makefile`、`.github/workflows/ci.yml`,Gitee Go 接入说明(见 §5.1)。
- **Prometheus `/metrics`**:各方法请求数与累计耗时。
- **Docker 改用 uv**:`uv sync --frozen` 按 `uv.lock` 安装,与开发/CI 同工具链、版本可复现。
- **可插拔结果缓存(默认关闭)**:`cache.py`(Null / LRU+TTL)、`OYE_RESULT_CACHE_SIZE/TTL_S`、请求级 `cache`(auto/refresh/off)、`from_cache` + `X-Cache` 头、`unload/reload` 失效、指标 `oye_cache_events_total`。`size=0` 时核心零开销。设计见 ADR 0003。

以下功能因需要交互式环境或外部资源,无法自动化测试,需人工验证:

## 4. 关键约定速查（实现时务必遵守)

- **坐标契约**:识别器只吃预处理图,吐基于**输入图**坐标的 `RawDetection`;偏移回映射与归一化统一在 `preprocessing.finalize_detections`,识别器内部绝不自己做。
- **重依赖懒加载**:`torch`/`paddle` 只在对应识别器首次推理时 import,顶层与 `--help`/轻量命令不触发。
- **测试用真实数据(强制)**:不得用 mock/fake 替代识别核心;HTTP/契约用真实 `TemplateRecognizer` 端到端,OCR/YOLO 用真实模型放 `tests/smoke/`(`make smoke`)。识别器抽象 + 依赖注入仍保留(便于注入真实组件)。详见 CLAUDE.md「测试用真实数据」。
- **训练隔离**:`training/` 独立目录,`src/` 任何文件**不得** import,有测试 `test_training_isolation.py` 守门。
- **统一错误契约**:成功无目标 = 200 + 空 detections(不是错误);失败才 4xx/5xx + `error_code`。
- **每任务 TDD**:写失败测试 → 跑红 → 最小实现 → 跑绿 → 提交,每个大任务至少 3 个小步骤逐步实现逐步测试。
- **报错必修**:存在任何报错(测试失败、lint/type 报错)必须修复后才允许 commit/push。
- **提交规范**:Conventional Commits,描述用简体中文(见 CLAUDE.md)。

## 5. 环境与常用命令

直接用 `uv`:

```bash
uv sync --extra dev                 # 安装开发依赖
uv sync --extra dev --extra ocr --extra yolo   # 装全量(含 OCR/YOLO 重依赖)
uv run pytest                       # 跑测试(默认跳过 smoke)
uv run pytest -m smoke              # 真实模型冒烟测试(需全量依赖)
uv run ruff check src tests         # lint
uv run mypy                         # 类型检查
uv run pre-commit run --all-files   # 全量质量门禁
uv run ocr-yolo serve               # 启动服务
```

或用 `make`(本地与 CI 共用快捷入口):

```bash
make install        # = uv sync --extra dev
make install-all    # 装全量依赖
make lint / format / type / test / smoke
make check          # 全量门禁(= bash scripts/check.sh,CI 入口)
```

> 所有 git 与项目命令都在 `MyProject/OcrYoloEngine/` 内执行;父级 `MyProject/.claude/` 不属于本仓库。

## 5.1 持续集成（CI）

**门禁单一事实来源:`scripts/check.sh`**(lint + 格式 + 类型 + 测试,任一失败即非零退出)。本地、CI 都跑它,口径一致。

- **GitHub(镜像场景)**:`.github/workflows/ci.yml` 已配置,push/PR 到 `master` 自动跑 `scripts/check.sh`。
- **Gitee Go(主库)**:在 Gitee 仓库「流水线」里新建流水线,执行步骤填 `bash scripts/check.sh` 即可(与上面等价)。
- 默认 CI 只跑快测(真实图像端到端,无需重模型);真实 OCR/YOLO 冒烟(`make smoke`)较重,按需在夜间/手动流水线跑。
