# 快速开始

> 目标：把服务跑起来，发一个请求拿到识别结果。
>
> 回到 [项目说明](项目说明.md) · 查接口怎么调 → [使用文档](使用文档.md) · 平台对接代码 → [接口集成指南](接口集成指南.md) · 部署到服务器 → [部署文档](部署文档.md)

---

## 前提

- Python ≥ 3.11
- [uv](https://docs.astral.sh/uv/)（Python 包管理工具）。没装就跑：
  ```bash
  curl -LsSf https://astral.sh/uv/install.sh | sh
  ```
  > Windows（PowerShell）：`powershell -c "irm https://astral.sh/uv/install.ps1 | iex"`
  >
  > 不想装 uv？也可以用 pip，见下面的「备选：用 pip 安装」。

---

## 1. 下载代码 + 装依赖

```bash
git clone https://gitee.com/xiaozai-van-liu/OcrYoloEngine.git
cd OcrYoloEngine
uv sync
```

基础安装只装网页服务 + 模板匹配的依赖，很快就好。

<details>
<summary>备选：用 pip 安装（不用 uv）</summary>

```bash
git clone https://gitee.com/xiaozai-van-liu/OcrYoloEngine.git
cd OcrYoloEngine
python3 -m venv .venv
source .venv/bin/activate          # Windows: .venv\Scripts\activate
pip install -e .
```

后续命令中所有 `uv run xxx` 替换成直接运行 `xxx`（确保虚拟环境已激活）。

</details>

> 对命令行不太熟的读者：
> - **没有 Python？** Ubuntu：`sudo apt install python3`；Windows：去 [python.org](https://www.python.org/downloads/) 下载安装，**勾选 "Add Python to PATH"**；macOS：`brew install python@3.12`
> - **没有 Git？** Ubuntu：`sudo apt install git`；Windows：去 [git-scm.com](https://git-scm.com/) 下载安装
> - **uv 装完不生效？** 关掉终端重新打开，或执行 `source $HOME/.local/bin/env`

## 2. 启动服务

```bash
uv run ocr-yolo serve
```

终端会打印 `浏览器打开 http://localhost:8000/docs 查看接口文档`——**直接点这个地址**就能在浏览器里看到接口文档页面。也可以直接访问 `http://localhost:8000`，会自动跳转。

如果 8000 端口被占用，会自动切换到下一个可用端口并提示。

> 这个终端窗口要一直开着（服务在前台运行）。后续操作**另开一个终端**，进入 OcrYoloEngine 目录。

验证：

```bash
curl http://localhost:8000/health
# 返回 {"status":"ok"} 就是正常的
```

## 3. 发第一个识别请求（模板匹配，开箱即用）

仓库自带了示例模板（一个小黑块图案 + 一张含有该图案的场景图），**不用准备任何文件**，装好就能试。

```bash
uv run python -c "
import base64, json, urllib.request

with open('tests/fixtures/golden_scene.png', 'rb') as f:
    b64 = base64.b64encode(f.read()).decode()

req = urllib.request.Request(
    'http://localhost:8000/v1/match',
    data=json.dumps({
        'image': {'base64': b64},
        'methods': ['template'],
        'templates': ['demo_block']
    }).encode(),
    headers={'Content-Type': 'application/json'},
)
resp = urllib.request.urlopen(req)
data = json.loads(resp.read())
det = data['method_results']['template']['detections'][0]
print(f'找到了！中心坐标: {det[\"center\"]}，把握程度: {det[\"confidence\"]:.2f}')
"
```

预期输出：`找到了！中心坐标: [90.0, 70.0]，把握程度: 0.99`

这个 `center` 坐标就是你的自动化脚本要点击的位置。

> **也可以用浏览器**：打开 `http://localhost:8000/docs`，找到 `/v1/match`，点"Try it out"试。

## 4. （可选）文字识别和目标检测

模板匹配适合找**固定不变的图标**。需要其他识别方式时，额外装对应的大包：

```bash
uv sync --extra ocr              # 文字识别（下载 PaddlePaddle，约 1-2GB）
uv sync --extra yolo             # 目标检测（下载 PyTorch，约 2-3GB）
uv sync --extra yolo --extra ocr # 全部都要
```

- **文字识别**：不用准备模型，首次使用自动下载。用 `/v1/ocr` 接口。
- **目标检测**：仓库自带通用模型 `yolov8n`（能识别人、车、动物等），用 `/v1/detect` 接口，`"model": "yolov8n"`。

## 5. （正式使用）配置你自己的模型和模板

上面的 demo 用的是仓库自带的示例配置（`configs/*.yaml.example`），装完直接能跑。要登记你自己的模型/模板时，**先复制一份实际配置再改**：

```bash
cp configs/models.yaml.example configs/models.yaml
cp configs/templates.yaml.example configs/templates.yaml
# 然后编辑 configs/models.yaml / configs/templates.yaml 写你自己的
```

实际的 `configs/models.yaml` / `configs/templates.yaml` 已被 git 忽略，改它不会和上游冲突、也不会误提交。具体格式见 [使用文档](使用文档.md#91-配置文件怎么用重要)。

## 常见问题

| 遇到什么 | 解决办法 |
|---|---|
| `uv: command not found` | 重开终端或执行 `source $HOME/.local/bin/env` |
| `ModuleNotFoundError: torch/paddle` | 跑 `uv sync --extra yolo --extra ocr` |
| `curl: command not found` | 装 curl 或直接用浏览器的 `/docs` 页面试 |
| `detections: []`（空的） | 这不是报错——没找到目标。可以调低 `conf_threshold` 再试 |
| `TEMPLATE_NOT_FOUND` | 模板名没登记。仓库自带的示例模板叫 `demo_block` |
| 端口被占用 | 服务会自动切换到下一个可用端口；也可手动指定：`uv run ocr-yolo serve --port 8001` |
| 安装大包时报错或卡住 | 网络问题，重跑一遍；或者先只用模板匹配（不需要大包） |
