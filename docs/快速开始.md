# 一、快速开始（5 分钟）

> 目标：把服务跑起来，发一个请求拿到识别结果。

## 前提

- Python ≥ 3.11
- [uv](https://docs.astral.sh/uv/)（Python 的包管理工具，比 pip 快很多）。没装就跑：
  ```bash
  curl -LsSf https://astral.sh/uv/install.sh | sh
  ```
  > 不想装 uv？也可以用 pip，见下面的「备选：用 pip 安装」。

## 1. 下载代码 + 装依赖

```bash
git clone https://gitee.com/xiaozai-van-liu/OcrYoloEngine.git
cd OcrYoloEngine
uv sync
```

基础安装只装网页服务 + 模板匹配的依赖，很快就好。

<details>
<summary>备选：用 pip 安装（不用 uv）</summary>

```bash
git clone https://gitee.com/xiaozai-van-liu/OcrYoloEngine.git
cd OcrYoloEngine
python3 -m venv .venv
source .venv/bin/activate          # Windows: .venv\Scripts\activate
pip install -e .
```

后续命令中所有 `uv run xxx` 替换成直接运行 `xxx`（确保虚拟环境已激活）。

</details>

## 2. 启动服务

```bash
uv run ocr-yolo serve
```

看到 `http://0.0.0.0:8000` 就是启动成功了。如果 8000 端口被占用，会自动切换到 8001、8002……并在终端提示实际使用的端口。浏览器打开 `http://localhost:8000/docs`（端口换成实际的）可以看到可视化的接口测试页面，能直接在网页上点击"Try it out"发请求。

验证一下：

```bash
curl http://localhost:8000/health
# 返回 {"status":"ok"} 就是正常的
```

## 3. 发第一个识别请求（模板匹配，开箱即用）

仓库自带了一套示例模板（一个小黑块图案 + 一张含有该图案的场景图），**不用你准备任何文件**，装好就能试。

用场景图做测试——在场景图里找小黑块模板：

```bash
# 用 Python 直接发请求（跨平台通用，推荐）
uv run python -c "
import base64, json, urllib.request

with open('tests/fixtures/golden_scene.png', 'rb') as f:
    b64 = base64.b64encode(f.read()).decode()

req = urllib.request.Request(
    'http://localhost:8000/v1/match',
    data=json.dumps({
        'image': {'base64': b64},
        'methods': ['template'],
        'templates': ['demo_block']
    }).encode(),
    headers={'Content-Type': 'application/json'},
)
resp = urllib.request.urlopen(req)
data = json.loads(resp.read())
det = data['method_results']['template']['detections'][0]
print(f'找到了！中心坐标: {det[\"center\"]}，把握程度: {det[\"confidence\"]:.2f}')
"
```

预期输出：`找到了！中心坐标: [90.0, 70.0]，把握程度: 0.99`

这就是模板匹配的完整流程：把截图发过来，告诉服务用哪个模板找，返回坐标。

> **也可以用 curl**（Linux/macOS）：
> ```bash
> B64=$(base64 -w0 tests/fixtures/golden_scene.png)  # macOS 改用 base64 -i
> curl -s http://localhost:8000/v1/match \
>   -H "Content-Type: application/json" \
>   -d "{\"image\": {\"base64\": \"$B64\"}, \"methods\": [\"template\"], \"templates\": [\"demo_block\"]}" \
>   | python3 -m json.tool
> ```
>
> **或者直接用浏览器**：打开 `http://localhost:8000/docs`，找到 `/v1/match`，点"Try it out"，把请求体粘贴进去就能试。

## 4. （可选）用文字识别——需要额外装包

文字识别（OCR）不需要你准备模型文件（首次使用会自动下载），但底层的深度学习框架比较大：

```bash
uv sync --extra ocr       # 会下载 PaddlePaddle，可能需要几分钟
```

装好后，用任意带文字的截图试：

```bash
uv run python -c "
import base64, json, urllib.request

with open('screenshot.png', 'rb') as f:     # 换成你的截图
    b64 = base64.b64encode(f.read()).decode()

req = urllib.request.Request(
    'http://localhost:8000/v1/ocr',
    data=json.dumps({'base64': b64}).encode(),
    headers={'Content-Type': 'application/json'},
)
resp = urllib.request.urlopen(req)
data = json.loads(resp.read())
for det in data['method_results']['ocr']['detections']:
    print(f'{det[\"text\"]}  把握 {det[\"confidence\"]:.2f}  坐标 {det[\"center\"]}')
"
```

> ⚠️ 注意：`/v1/ocr` 的请求体直接是 `{"base64": "..."}`，不需要外面套一层 `image`。其他接口（detect/match/recognize）需要写成 `{"image": {"base64": "..."}, ...}`。

## 5. （可选）用目标检测 YOLO

仓库自带了一个通用检测模型 `yolov8n`（能识别人、车、动物等常见物体），需要额外装 YOLO 的大包：

```bash
uv sync --extra yolo      # 会下载 PyTorch，可能需要几分钟，磁盘占 2-3GB
```

装好后试一下（用任意照片，最好有人/车/动物之类的常见物体）：

```bash
uv run python -c "
import base64, json, urllib.request

with open('photo.jpg', 'rb') as f:         # 换成你的照片
    b64 = base64.b64encode(f.read()).decode()

req = urllib.request.Request(
    'http://localhost:8000/v1/detect',
    data=json.dumps({
        'image': {'base64': b64},
        'methods': ['yolo'],
        'model': 'yolov8n'
    }).encode(),
    headers={'Content-Type': 'application/json'},
)
resp = urllib.request.urlopen(req)
data = json.loads(resp.read())
for det in data['method_results']['yolo']['detections']:
    print(f'{det[\"label\"]}  把握 {det[\"confidence\"]:.2f}  坐标 {det[\"center\"]}')
"
```

> 自带的 `yolov8n` 是通用模型。如果要检测特定物体（比如游戏里的角色），需要自己训练模型，放到 `models_store/` 目录，在 `configs/models.yaml` 里登记。训练指南见 `training/README.md`。

全部接口、字段说明、错误码见 [使用文档](使用文档.md)；部署到服务器 / 用 Docker 见 [部署文档](部署文档.md)；想搞懂它内部怎么工作的见 [项目说明](项目说明.md)。

# 二、从零开始（小白手把手）

> 假设你**完全没用过这个项目**，对命令行也不太熟。跟着一步步来就行。
> 已经熟悉命令行 → 直接看上面的「一、快速开始」就够了。

以 **Ubuntu / Linux** 为例，Windows / macOS 的不同之处会在每步标注。

## 第 0 步：先搞清楚这个服务在干什么

你有一个自动化测试脚本（比如用来自动测试 App 的脚本），它运行时会截屏。

它的需求是：**截图里某个按钮/文字在屏幕上的什么位置**。

本服务就是帮它"看图"的——你把截图发过来，它告诉你目标在哪个坐标。**它只负责看，不负责点击**——点击还是你的脚本自己做。

使用顺序：**先在某台机器上把服务跑起来 → 你的脚本把截图发给它 → 它返回坐标 → 你的脚本拿坐标去点击**。

## 第 1 步：确保有 Python（3.11 以上）

先看看装没装：

```bash
python3 --version
```

如果显示 `Python 3.11.x` 或更高，跳过本步。否则：

- Ubuntu：`sudo apt update && sudo apt install -y python3 python3-venv`
- Windows：去 [python.org](https://www.python.org/downloads/) 下载安装，安装时**一定要勾选 "Add Python to PATH"**。
- macOS：`brew install python@3.12`

## 第 2 步：装 uv（依赖管理工具）

`uv` 是用来安装项目依赖和运行命令的工具，比传统的 pip 快很多。

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

装完后它会提示你需要刷新一下终端环境。**关掉终端重新打开**，或者执行：

```bash
source $HOME/.local/bin/env    # 或者直接重开终端
uv --version                   # 能打印出版本号就说明装好了
```

> Windows 用户（PowerShell）：`powershell -c "irm https://astral.sh/uv/install.ps1 | iex"`

<details>
<summary>不想装 uv？用 pip 也行</summary>

后面的步骤中，把 `uv sync` 换成 `pip install -e .`，把 `uv run xxx` 换成直接运行 `xxx`（需要先激活虚拟环境）：

```bash
python3 -m venv .venv
source .venv/bin/activate     # Windows: .venv\Scripts\activate
pip install -e .
# 之后直接用 ocr-yolo serve、ocr-yolo infer 等命令
```

</details>

## 第 3 步：把代码下载下来

```bash
git clone https://gitee.com/xiaozai-van-liu/OcrYoloEngine.git
cd OcrYoloEngine
```

后面所有命令都在这个 `OcrYoloEngine` 目录里执行。

## 第 4 步：安装依赖

```bash
uv sync
```

基础安装很快（大约几秒到半分钟），装好后就能用模板匹配功能了。

> 后面要用文字识别或目标检测，再回来装额外的包就行，见第 8 步。

## 第 5 步：启动服务

```bash
uv run ocr-yolo serve
```

看到类似 `Uvicorn running on http://0.0.0.0:8000` 就是成功了（如果 8000 被占用，会自动换一个端口并提示你）。**这个终端窗口要一直开着**（服务在前台运行）。

验证（**另开一个终端**，进入 OcrYoloEngine 目录）：

```bash
curl http://localhost:8000/health
```

返回 `{"status":"ok"}` 就说明一切正常。也可以直接用浏览器打开 `http://localhost:8000/docs`，会看到一个可视化的接口测试页面，可以直接在网页上试着发请求。

## 第 6 步：发第一个识别请求（模板匹配）

仓库自带了示例素材，**不用准备任何文件**，安装完就能试。

**另开一个终端**，在 `OcrYoloEngine` 目录下运行：

```bash
uv run python -c "
import base64, json, urllib.request

# 读取仓库自带的测试场景图（白底上有一个小黑块）
with open('tests/fixtures/golden_scene.png', 'rb') as f:
    b64 = base64.b64encode(f.read()).decode()

# 发请求：在场景图里找 demo_block 模板
req = urllib.request.Request(
    'http://localhost:8000/v1/match',
    data=json.dumps({
        'image': {'base64': b64},
        'methods': ['template'],
        'templates': ['demo_block']
    }).encode(),
    headers={'Content-Type': 'application/json'},
)
resp = urllib.request.urlopen(req)
data = json.loads(resp.read())

# 看结果
det = data['method_results']['template']['detections'][0]
print(f'找到了！中心坐标: {det[\"center\"]}，把握程度: {det[\"confidence\"]:.2f}')
"
```

预期输出：

```
找到了！中心坐标: [90.0, 70.0]，把握程度: 0.99
```

🎉 **恭喜，整个流程跑通了！**

这个 `center` 坐标（`[90.0, 70.0]`）就是你的自动化脚本要点击的位置。`confidence` 是把握程度（0.99 表示 99% 有把握是这个图标）。

## 第 7 步：在你自己的脚本里调用

实际使用时，你的 Python 脚本大概长这样：

```python
import base64
import json
import urllib.request

def find_on_screen(screenshot_path, template_name):
    """在截图里找指定的模板图标,返回中心坐标。"""
    with open(screenshot_path, "rb") as f:
        b64 = base64.b64encode(f.read()).decode()

    req = urllib.request.Request(
        "http://localhost:8000/v1/match",
        data=json.dumps({
            "image": {"base64": b64},
            "methods": ["template"],
            "templates": [template_name],
        }).encode(),
        headers={"Content-Type": "application/json"},
    )
    resp = urllib.request.urlopen(req)
    data = json.loads(resp.read())

    detections = data["method_results"]["template"]["detections"]
    if not detections:
        return None  # 没找到
    return detections[0]["center"]  # 返回 [x, y]


# 用法
pos = find_on_screen("screenshot.png", "demo_block")
if pos:
    x, y = pos
    print(f"找到了，去点击 ({x}, {y})")
    # your_automation_framework.click(x, y)
else:
    print("没找到")
```

> 这段代码只用了 Python 标准库，不需要额外装 requests 之类的包。如果你项目里已经有 requests 库，也可以用 `requests.post(...)` 发请求，更简洁。

## 第 8 步（可选）：装文字识别和目标检测

模板匹配适合找**固定不变的图标**。如果你需要：

- **识别文字**（比如截图里的按钮文字、弹窗文字）→ 装文字识别（OCR）：
  ```bash
  uv sync --extra ocr
  ```
  会下载 PaddlePaddle 深度学习框架，**大约 1-2GB，需要几分钟**。装好后用 `/v1/ocr` 接口即可。

- **检测特定物体**（比如游戏里的角色、道具）→ 装目标检测（YOLO）：
  ```bash
  uv sync --extra yolo
  ```
  会下载 PyTorch 框架，**大约 2-3GB，需要几分钟**。仓库自带了一个通用检测模型 `yolov8n`（能识别人、车、动物等），用 `/v1/detect` 接口时 `"model": "yolov8n"` 即可。

- **全部都要**：
  ```bash
  uv sync --extra yolo --extra ocr
  ```

> ⚠️ 安装这些大包需要磁盘空间（总共约 3-5GB）和稳定的网络。如果安装中途断了，重新跑一遍命令就行。

## 常见问题

| 遇到什么 | 原因和解决办法 |
|---|---|
| `uv: command not found` | 第 2 步的路径没生效，重开终端或者执行 `source $HOME/.local/bin/env` |
| 启动报 `ModuleNotFoundError: torch/paddle` | 没装完整版依赖，跑一下 `uv sync --extra yolo --extra ocr` |
| `curl: command not found` | 装一下 curl：`sudo apt install curl`；或者直接用浏览器的 `/docs` 页面试 |
| 请求返回 `detections: []`（空的） | **这不是报错**——只是没找到达到要求的目标。可以把 `conf_threshold`（把握程度要求）调低一点再试 |
| 返回 `{"error_code":"INVALID_IMAGE"}` | 图片有问题——可能路径不对、文件损坏、或者编码转换不完整 |
| 返回 `{"error_code":"MODEL_NOT_FOUND"}` | 你指定的模型名没有在 `configs/models.yaml` 里登记，或者模型文件没放对位置 |
| 返回 `{"error_code":"TEMPLATE_NOT_FOUND"}` | 你指定的模板名没有在 `configs/templates.yaml` 里登记。仓库自带的示例模板叫 `demo_block` |
| 返回 `{"error_code":"PATH_NOT_ALLOWED"}` | 用本地路径传图时，路径不在允许的目录里。需要设置环境变量：`OYE_ALLOWED_PATH_ROOTS='["/你的截图目录"]'`（JSON 数组格式） |
| 端口被其他程序占用了 | 服务会自动切换到下一个可用端口并提示；也可以手动指定：`uv run ocr-yolo serve --port 8001` |
| 安装 paddle/torch 时报错或卡住 | 可能是网络问题或平台兼容性。试试换国内镜像：`pip install paddlepaddle -i https://mirror.baidu.com/pypi/simple`；或者先只用模板匹配（不需要这些大包） |
