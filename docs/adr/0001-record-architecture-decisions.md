# 0001 - 使用架构决策记录（ADR）

- 状态：已接受
- 日期：2026-06-03
- 决策者：@vincent

## 背景与问题

项目需要一种轻量、可追溯的方式来记录重要的架构决策及其背后的原因，避免日后无人记得「当初为什么这么选」。

## 考虑过的方案

- 不专门记录，靠提交信息和记忆
- 在 Wiki 中记录
- 在仓库内使用 Markdown 形式的 ADR（MADR）

## 决策结果

选择在仓库内使用 [MADR](https://github.com/adr/madr) 风格的架构决策记录，存放于 `docs/adr/`。决策与代码同仓库、同版本管理，便于随代码评审一起审阅。

### 正面影响

- 决策与代码一起进行版本管理与评审
- 新成员可快速了解关键决策的来龙去脉

### 负面影响 / 代价

- 需要团队养成在做重要决策时撰写记录的习惯

## 备注

新增记录时复制 [`template.md`](template.md)，编号递增。
