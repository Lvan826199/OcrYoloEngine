# 0002 - 模板匹配采用归一化 SAD 置信度

- 状态：已接受
- 日期：2026-06-03
- 决策者：@vincent

## 背景与问题

模板匹配识别器需要为每个匹配位置输出一个 0~1 的置信度。设计稿最初采用 OpenCV 的 `cv2.matchTemplate(..., TM_CCOEFF_NORMED)`（归一化互相关）。但实测发现:**当模板是常量（纯色）图像时,归一化互相关的方差为零、数学上退化**,OpenCV 对场景中所有常量区域(包括纯色背景)都返回 1.0,导致海量误匹配。

而本项目的核心场景之一正是**手机游戏 UI**,其中纯色按钮、纯色图标非常常见,这个退化问题会实际发生。

## 考虑过的方案

- `TM_CCOEFF_NORMED`(归一化互相关):光照不变性好,但对常量模板退化,误匹配严重。
- `TM_SQDIFF_NORMED`(归一化平方差):对常量模板分母为零,可能产生 NaN,同样不稳健。
- **归一化 SAD/SSD**:用 `TM_SQDIFF` 得平方差,再按最大可能差值 `255² × 模板像素数` 归一化为 `1.0 - diff / max_diff`。逐像素完全一致得 1.0,最大差异得 0.0。

## 决策结果

采用**归一化 SAD 置信度**:`score = 1.0 - matchTemplate(scene, tmpl, TM_SQDIFF) / (255² × tmpl.size)`。对任意模板(含纯色)都稳健,置信度语义清晰且恒落在 [0, 1]。多尺度匹配 + 阈值过滤 + NMS 去重的整体流程不变。

实现见 `src/ocr_yolo_engine/recognizers/template.py`。

### 正面影响

- 对纯色/常量模板稳健,契合游戏 UI 场景。
- 置信度物理含义直观(像素差异程度),阈值好调。
- 避免 NaN / 全图误匹配。

### 负面影响 / 代价

- 绝对像素差异对**全局光照变化**的鲁棒性弱于归一化互相关。后续若遇到强光照变化场景,可在 `params` 中按模板选择打分方式,或引入特征点匹配(ORB/SIFT)作为补充。
