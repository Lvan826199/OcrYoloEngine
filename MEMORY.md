# 项目记忆（跨机共享）

> 本文件由 CLAUDE.md 通过 `@MEMORY.md` 自动加载，随 git 在多台电脑间共享。
> **定位**：放「跨会话/跨机器都该记住、但不属于正式文档」的工作偏好与注意事项。
> 项目事实（进度/约定/修复记录）以 `docs/` 为权威，不在这里重复。
> **维护**：新增重要记忆追加到对应小节，过时的直接删除；本文件是共享记忆的唯一权威，
> 本机 `~/.claude` 持久 memory 只放机器特定内容（路径、环境差异等）。

## 用户工作偏好

- **自主执行**：按 Claude 的判断直接推进，不要反复弹确认；只在破坏性操作或真正的范围变更时询问。
- **子代理模型**：子代理一律用 Opus 4.8（`model: opus`），禁止切换到其他模型。
- **权限持久化**：已授权过的权限直接写进 `.claude/settings.local.json`，不再重复询问。

## 跨会话注意事项

- 修复/优化批次严格按 CLAUDE.md「Bug 修复策略」执行：计划入 `plan/`，日志记 `docs/bug修复日志.md`（倒序追加）。
- 「有意不改」的已知决策（如 `/ready` 语义、同步路由双线程占用）记录在 `docs/bug修复日志.md`，勿当新发现重复评估。
- `tests/fixtures/*.expected.json` 期望值必须出自真实运行，用 `scripts/gen_expected.py` 生成草稿后人工审定容差。
- 游戏 UI 模板匹配的使用经验：模板紧贴图标裁剪、少含背景；深色/低对比模板把 threshold 提到 0.95+（详见使用文档 9.3）。
